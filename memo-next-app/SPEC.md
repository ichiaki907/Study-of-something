# memo-next-app 仕様整理

## 1. アプリ概要
- 本アプリは Next.js（App Router）+ Chakra UI で構築された、1画面の簡易メモアプリ。
- ユーザーは入力欄にテキストを入力し、`更新` ボタンで表示領域へ反映する。
- `クリア` ボタンで入力中テキストと表示中テキストを同時に空にする。

## 2. 技術仕様
- フレームワーク: Next.js（`app/` ディレクトリ構成）
- UI: Chakra UI（`ChakraProvider` + `CacheProvider`）
- レンダリング: `page.tsx` は `"use client"` を使用したクライアントコンポーネント
- 言語: TypeScript / TSX

## 3. 画面仕様
### 3.1 レイアウト
- 画面全体に白背景を設定
- 中央カラム（最大幅 `640px`）にメモ入力 UI を配置
- 中央カラムは薄いグレー背景・上下余白あり

### 3.2 表示要素
1. 見出し: `メモ帳`
2. 複数行入力欄（`Textarea`）
3. `更新` ボタン
4. `クリア` ボタン
5. 表示領域（現在の保存メモを表示）

## 4. 状態管理仕様
`Home` コンポーネントでは次の 2 状態を保持する。

- `value: string`
  - 入力欄（`Textarea`）の現在値
- `memo: string`
  - 表示領域に確定表示する値

### 4.1 状態遷移
- 入力欄変更 (`onChange`)
  - `value` のみ更新
  - `memo` は変更しない
- `更新` クリック
  - `memo = value`
- `クリア` クリック
  - `value = ""`
  - `memo = ""`

## 5. イベント仕様
- 入力イベント: `React.ChangeEvent<HTMLTextAreaElement>` を受け取り、`event.target.value` を `value` に反映
- 更新イベント: 現在の `value` を `memo` に同期
- クリアイベント: `value` と `memo` を同時に初期化

## 6. 現在の挙動（UX観点）
- 入力中は表示領域は変化しない（`更新` 押下までは確定しない）
- `更新` 押下時にのみ表示領域へ反映される
- `クリア` 押下で入力欄・表示領域がともに空になる

## 7. 非機能・改善ポイント（メモ）
- 現状は単一コンポーネント構成で、入力欄更新のたびに `Home` 全体が再レンダーされる。
- 表示領域を子コンポーネントへ分離すると、`React.memo` の効果検証がしやすい。
- ただし最適化は、描画コスト計測（Profiler 等）後に導入判断する。
